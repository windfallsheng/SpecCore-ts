## 1. 会议室浏览与搜索 — P0

**用户故事**: 作为员工，我希望在手机上快速浏览可用会议室，以便当场预订。

**BDD 验收标准**:

```
Given 员工打开 H5 首页
When  页面加载完成
Then  看到可用会议室卡片列表（名称/容量/设备/剩余时段）

Given 员工在会议室列表页面
When  选择楼层筛选"2层"，容量筛选"10-20人"
Then  列表仅显示 2 层 10-20 人的会议室，已满的置底
```

| GET | /api/h5/rooms | 可用会议室列表 |
| GET | /api/h5/rooms/:id | 详情 + 今日时间轴 |

---

## 2. 快速预订 — P0

**用户故事**: 作为员工，我希望选择时段后自动匹配可用会议室完成预订。

**BDD 验收标准**:

```
Given 员工打开会议室详情
When  选择日期、开始时间 10:00、结束时间 11:30，输入主题，点击预订
Then  系统检测无冲突后创建预订，推送成功通知

Given 员工选择时段与已有预订冲突
When  点击预订
Then  系统提示冲突，推荐最近可用时段

Given 员工在"我的预订"列表中
When  会议开始前 30 分钟以上，点击取消
Then  预订取消成功。若不足 30 分钟则拒绝取消
```

| POST | /api/h5/bookings | 创建预订（冲突自动检测） |
| GET | /api/h5/bookings/mine | 我的预订列表 |
| DELETE | /api/h5/bookings/:id | 取消（开始前 30min 可取消） |

---

## 3. 会议签到 — P1

**用户故事**: 作为参会人，我希望扫码签到，以便记录实际参会情况。

**BDD 验收标准**:

```
Given 员工在签到时间窗内（开始前15分钟~开始后30分钟）
When  扫描会议室二维码或手动点击签到
Then  签到成功，状态更新为"已签到"

Given 会议开始 30 分钟后仍未签到
When  系统定时任务执行
Then  该预订自动标记为"缺席"
```

| POST | /api/h5/checkin/:bookingId | 会议签到 |
| GET | /api/h5/checkin/:bookingId | 签到状态 |

---

## 4. 消息通知 — P1

**用户故事**: 作为用户，我希望及时收到预订状态变更通知。

**BDD 验收标准**:

```
Given 用户的预订被管理员取消
When  刷新通知列表
Then  看到一条未读通知"您的预订已被取消"，红色角标数+1

Given 用户点击通知
Then  跳转到对应预订详情，通知标记为已读
```

| GET | /api/h5/notifications | 通知列表 |
| PUT | /api/h5/notifications/:id/read | 标记已读 |

## 术语表

| 术语 | 说明 |
| :--- | :--- |
| 冲突检测 | 同一会议室同一时段不允许重复预订 |
| 签到时间窗 | 会议开始前 15 分钟 ~ 开始后 30 分钟 |
| BDD | 行为驱动开发，用 Given/When/Then 描述验收标准 |
