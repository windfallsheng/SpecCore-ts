# 后台管理端 — Web 管理端需求

> **平台**: Web 管理端 | Vue 3 + Element Plus | PC 浏览器  
> **原型参考**: `../prototype-admin.html`

---

## 功能优先级

| 优先级 | 功能 | 说明 |
| :--- | :--- | :--- |
| **P0** | 会议室管理 CRUD | 核心业务，无此不可上线 |
| **P0** | 预订管理 | 核心业务 |
| **P0** | 用户权限管理（RBAC） | 安全基线 |
| **P1** | 数据统计看板 | 运营需要，可延后一个迭代 |
| **P2** | 批量导入会议室 | 锦上添花 |

## 范围边界

| 范围内 | 范围外 |
| :--- | :--- |
| 会议室增删改查 | 会议室三维平面图 |
| 预订管理（查看+取消） | 预订新建（属于 H5 端） |
| 用户角色权限管理 | 组织架构同步（依赖 HR 系统） |
| 数据统计看板 | 自定义报表引擎 |

## 依赖关系

| 依赖 | 说明 |
| :--- | :--- |
| 会议室管理服务 | 提供会议室 CRUD 接口 |
| 预订订单服务 | 提供预订查询和取消接口 |
| 统一认证中心 | JWT 登录 + RBAC 鉴权 |

## 1. 会议室管理（CRUD） — P0

**用户故事**: 作为管理员，我希望增删改查会议室信息，以便统一管理所有会议室资源。

**BDD 验收标准**:

```
Given 管理员已登录系统
When  点击"新增会议室"，填写名称/容量/楼层/设备并提交
Then  会议室列表中出现新记录，状态为"空闲"

Given 管理员在会议室列表页面
When  选择楼层筛选"3层"，并搜索"泰山"
Then  列表仅显示 3 层包含"泰山"的会议室

Given 管理员在会议室列表页面
When  点击某会议室的"删除"按钮并确认
Then  该会议室状态变更为"已删除"，列表不再显示
```

| POST | /api/admin/rooms | 新增会议室 |
| GET | /api/admin/rooms?page=1&floor=3&kw=泰山 | 分页列表 |
| GET | /api/admin/rooms/:id | 详情 |
| PUT | /api/admin/rooms/:id | 编辑 |
| DELETE | /api/admin/rooms/:id | 软删除 |

## 2. 预订管理 — P0

**用户故事**: 作为管理员，我需要查看和管控所有会议预订，以便处理违规占用。

**BDD 验收标准**:

```
Given 管理员在预订管理页面
When  筛选日期"今天"和状态"进行中"
Then  列表仅显示今天正在进行中的预订

Given 管理员查看某条预订详情
When  点击"取消预订"，填写原因"会议室被外部客户占用"
Then  预订状态变为"已取消"，预订人收到通知
```

| GET | /api/admin/bookings | 列表（按日期/会议室/状态筛选） |
| GET | /api/admin/bookings/:id | 详情 |
| DELETE | /api/admin/bookings/:id | 强制取消（需原因） |

---

## 3. 用户权限管理（RBAC） — P0

**用户故事**: 作为超级管理员，我希望管理用户及其角色权限，以便控制系统访问。

**BDD 验收标准**:

```
Given 超级管理员在用户列表页面
When  将用户"张三"的角色从"user"改为"admin"
Then  张三下次登录获得管理员权限

Given 管理员在用户列表页面
When  点击用户"李四"的"禁用"按钮
Then  李四无法登录系统，已有的预订不受影响
```

| POST | /api/admin/users | 新增用户 |
| GET | /api/admin/users | 列表（按角色/部门筛选） |
| PUT | /api/admin/users/:id | 编辑信息 |
| PUT | /api/admin/users/:id/role | 变更角色 |
| DELETE | /api/admin/users/:id | 禁用/启用 |

---

## 4. 数据统计看板 — P1

**用户故事**: 作为管理员，我需要查看使用数据报表，以便优化资源配置。

**BDD 验收标准**:

```
Given 管理员打开仪表盘
When  页面加载完成
Then  显示 4 张统计卡片（总会议室/今日预订/占用率/本月预订）及趋势图
```

| GET | /api/admin/dashboard/stats | 核心统计 |
| GET | /api/admin/dashboard/trends | 近30天趋势 |
| GET | /api/admin/dashboard/hot-rooms | TOP10 排行 |

## 术语表

| 术语 | 说明 |
| :--- | :--- |
| 软删除 | 标记删除状态而非物理删除，数据可恢复 |
| RBAC | 基于角色的访问控制（Role-Based Access Control） |
| P0/P1/P2 | 优先级：P0 必须上线，P1 应该上线，P2 锦上添花 |
