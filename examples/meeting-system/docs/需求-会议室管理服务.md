# 会议室管理服务 — 后台服务需求

> **服务名称**: room-service  
> **技术栈**: Spring Boot 3 + MyBatis-Plus + MySQL  
> **职责**: 会议室资源 CRUD、设备管理、楼层管理、日程查询

---

## 功能优先级

| 优先级 | 功能 | 说明 |
| :--- | :--- | :--- |
| **P0** | 会议室 CRUD | 核心 API |
| **P0** | 会议室列表（分页+筛选） | 前端必需 |
| **P1** | 会议室日程查询 | 前端可选 |

## 范围边界

| 范围内 | 范围外 |
| :--- | :--- |
| 会议室数据 CRUD | 预订业务逻辑（属 booking-service） |
| 用户数据管理 | 用户认证鉴权（属统一认证中心） |
| 提供 REST API | 消息推送 |

## 依赖关系

| 依赖 | 说明 |
| :--- | :--- |
| MySQL 8.0 | 数据持久化 |
| 统一认证中心 | JWT Token 校验 |

---

## 接口清单

| 方法 | 路径 | 说明 | 优先级 |
| :--- | :--- | :--- | :--- |
| POST | /api/rooms | 新增会议室 | P0 |
| GET | /api/rooms | 会议室列表（分页+筛选） | P0 |
| GET | /api/rooms/:id | 会议室详情 | P0 |
| PUT | /api/rooms/:id | 更新会议室 | P0 |
| DELETE | /api/rooms/:id | 软删除会议室 | P0 |
| GET | /api/rooms/:id/schedule | 今日时间安排 | P1 |

## BDD 验收标准

```
Given 会议室表中有 12 条数据
When  请求 GET /api/rooms?page=1&size=10
Then  返回第 1 页 10 条数据，total=12

Given 请求新增会议室，名称为空
When  POST /api/rooms { "name": "" }
Then  返回 400，提示"名称不能为空"

Given 请求删除不存在的会议室
When  DELETE /api/rooms/999
Then  返回 404，提示"会议室不存在"
```

## 数据库：rooms 表

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BIGINT | 主键自增 |
| name | VARCHAR(100) | 会议室名称 |
| capacity | INT | 容纳人数 |
| floor | VARCHAR(20) | 所在楼层 |
| equipment | JSON | 设备列表（投影仪/白板/视频会议/音响/电话） |
| status | TINYINT | 0=空闲 1=使用中 2=维护中 |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 更新时间 |

## 数据库：users 表

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| id | BIGINT | 主键自增 |
| name | VARCHAR(50) | 姓名 |
| email | VARCHAR(100) | 邮箱（登录账号） |
| role | VARCHAR(20) | super_admin/admin/user |
| status | TINYINT | 0=正常 1=禁用 |
| created_at | DATETIME | 创建时间 |

---

## 非功能要求

| 类别 | 要求 |
| :--- | :--- |
| 性能 | 列表查询 < 200ms |
| 并发 | 支持 500 TPS |
| 安全 | JWT 认证 + RBAC 权限校验 |
| 日志 | 所有写操作记录审计日志 |

## 术语表

| 术语 | 说明 |
| :--- | :--- |
| TPS | 每秒事务数（Transactions Per Second） |
| 软删除 | 标记删除状态，数据不物理清除 |
| MyBatis-Plus | MyBatis 增强工具，简化 CRUD |
