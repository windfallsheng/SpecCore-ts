# Task-001 任务CRUD - 后端需求

> 创建时间：2026-07-14 | 任务类型：feature | 关联期次：Q3 | 关联需求：REQ-001

## 1. 需求背景

用户需要能够创建、查看、编辑、删除任务。这是任务管理系统的核心功能，后续的「任务分配」和「任务筛选」都依赖于此。

## 2. 功能描述

- **创建任务**：输入标题、描述、优先级（HIGH/MEDIUM/LOW）、截止日期，创建一条新任务
- **查看列表**：分页展示所有任务，默认 20 条/页，支持按优先级排序
- **查看详情**：点击任务行展开完整信息，包括创建时间和修改时间
- **编辑任务**：修改标题、描述、优先级、截止日期，记录修改日志
- **删除任务**：软删除（is_deleted = true），可通过管理员工具恢复

## 3. 接口定义

详见 _shared/API_CONTRACT.yaml

| 方法 | 路径 | 说明 |
| :--- | :--- | :--- |
| POST | /api/v1/tasks | 创建任务 |
| GET | /api/v1/tasks?page=1&size=20 | 分页列表 |
| GET | /api/v1/tasks/{id} | 查看详情 |
| PUT | /api/v1/tasks/{id} | 更新任务 |
| DELETE | /api/v1/tasks/{id} | 软删除 |

## 4. 验收标准

- [ ] POST 创建成功返回 201 + 任务 ID，参数校验失败返回 400
- [ ] GET 列表返回分页结构 { total, page, data: [...] }
- [ ] GET 详情返回完整对象，不存在的 ID 返回 404
- [ ] PUT 仅更新传入字段（PATCH 语义），成功返回 200
- [ ] DELETE 软删除成功返回 204，已删除 ID 返回 404
- [ ] 优先级必须为 HIGH / MEDIUM / LOW 之一
- [ ] 单元测试覆盖率 > 80%
