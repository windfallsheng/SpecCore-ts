# SpecCore 场景实战指南

> 从真实开发场景出发，逐步演示 SpecCore 的完整使用流程。

---

## 场景一：从零启动新项目

```bash
# 1. 初始化
cd my-project
speccore init

# 2. 导入已有代码到全量层
speccore import --project=backend --path=./src --type=backend
speccore import --project=frontend --path=./web --type=web

# 3. 创建第一个期次
speccore iteration create --name=期次-2026-07 --goal="用户认证系统"

# 4. 创建任务
speccore new-task --name="用户登录" --platforms=web,h5 --type=feature
speccore new-task --name="用户注册" --platforms=web,h5 --type=feature
speccore new-task --name="密码重置" --platforms=web --type=feature

# 5. 查看进度
speccore progress --detail
```

---

## 场景二：日常开发执行

```bash
# 1. 查看任务计划和依赖
speccore plan --iteration=期次-2026-07

# 2. 选择一个任务开始开发（自动创建 Git 分支）
speccore execute --task=Task-001 --force
# → 🌿 Created branch: feature/Task-001-user-login

# 3. 查看当前分支关联
speccore current
speccore current --commit    # 生成 commit 消息

# 4. 在代码中打 @spec 标注
# // @spec Task-001-user-login
# @RestController
# public class AuthController { ... }

# 5. 反向同步（代码 → Spec）
speccore sync --dry-run       # 预览
speccore sync                 # 写入 TASK.md

# 6. 查看追溯链
speccore trace --task=Task-001

# 7. 标记完成
speccore update --task=Task-001 --status=completed

# 8. 归档
speccore archive --task=Task-001
```

## 场景三：Bug 修复流程

```bash
# 1. 创建 Bug 修复任务
speccore bugfix --name="登录超时" --desc="Token 过期导致 401"

# 2. 开始修复
speccore execute --task=Task-004 --force

# 3. 修复代码 + @spec 标注
# // @spec Task-004-login-timeout
# if (token.isExpired()) refreshToken();

# 4. 反向同步
speccore sync

# 5. 变更记录
speccore change --req=REQ-001 --desc="修复 token 过期逻辑" --task=Task-004          

# 6. 完成
speccore update --task=Task-004 --status=completed
```

## 场景四：多人协作

```bash
# 1. 架构师创建期次和任务
speccore iteration create --name=期次-2026-07
speccore new-task --name="用户登录" --platforms=web,h5
speccore new-task --name="用户注册" --platforms=web,h5

# 2. 后端开发领取任务
git checkout -b feature/Task-001 # 或 speccore execute --task=Task-001 --force
speccore update --task=Task-001 --assignee=张三
speccore execute --task=Task-001 --force

# 3. 前端开发领取任务
git checkout -b feature/Task-002
speccore update --task=Task-002 --assignee=李四
speccore execute --task=Task-002 --force

# 4. 查看团队进度
speccore progress --detail
speccore health

# 5. 变更影响分析
speccore impact --req=REQ-001
# → 提示：该需求变更会影响 Task-001, Task-002

# 6. 生成交接文档
speccore handover --iteration=期次-2026-07
```

## 场景五：需求变更处理

```bash
# 1. 分析变更影响
speccore impact --req=REQ-001
# → 影响范围: Task-001(用户登录), Task-002(用户注册)

# 2. 创建基线（变更前快照）
speccore baseline --name=before-change

# 3. 描述变更
speccore change --req=REQ-001 --desc="登录增加短信验证码"

# 4. 更新受影响的 Task
speccore update --task=Task-001 --status=in_progress

# 5. 对比变更前后
speccore diff --source=before-change --target=current

# 6. 验证
speccore validate --iteration=期次-2026-07
```

## 场景六：长任务链分批执行

```bash
# 有 12 个 Task，每 3 个一批
speccore execute --all --batch-size=3 --dry-run    # 预览分批方案
speccore execute --all --batch-size=3 --force      # 执行

# 执行到第 2 批中断了...
# → 从第 3 批继续
speccore execute --resume

# 交互式：手动选择哪些任务执行
speccore execute --all --interactive
```

## 场景七：CI/CD 集成

```yaml
# .github/workflows/speccore-ci.yml
name: SpecCore Validate

on: [push, pull_request]

jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install SpecCore
        run: npm install -g speccore
      - name: Validate Specs
        run: speccore validate --full
      - name: Check Health
        run: speccore health
      - name: Progress Report
        run: speccore progress --format=json > progress.json
```

```bash
# 本地安装 Git hooks
speccore hooks install
# → pre-commit: 检查 @spec 标注
# → pre-push: 运行 speccore validate
```

## 场景八：全量层管理

```bash
# 1. 查看组织级需求全貌
speccore global-status

# 2. 从全量需求池拉取需求生成期次
speccore iteration-from-global --reqs=REQ-001,REQ-002,REQ-003 --name=期次-2026-Q3

# 3. 开发完成后同步回全量层
speccore sync-global --iteration=期次-2026-Q3

# 4. 需求变更历史追踪
speccore history --req=REQ-001

# 5. 定期重建索引
speccore index-update
```

## 场景九：Shell 项目迁移

```bash

npm install -g speccore

cd old-speccore-project
speccore migrate --dry-run

speccore validate
```

## 场景十一：项目健康检查

```bash
# 1. 整体健康度
speccore health

# 2. 按平台查看进度
speccore progress --platform=web --detail
speccore progress --platform=h5 --detail

# 3. 生成报告
speccore report --format=html --output=report.html --team --risk

# 4. 可视化仪表盘
speccore dashboard --output=dashboard.html
```

## 场景十二：建立项目基线

```bash
# 1. 发版前创建基线
speccore baseline --name=v2.0-release

# 2. 继续开发 v2.1...

# 3. 对比版本差异
speccore diff --source=v2.0-release --target=期次-2026-07

# 4. 需要时回滚需求
speccore baseline --restore=v2.0-release --req=REQ-005

# 5. 查看基线列表
speccore baseline --list
```

## 快速参考

| 场景 | 第一步 | 核心命令 |
| :--- | :--- | :--- |
| 新项目 | `speccore init` | `init → import → iteration create → new-task` |
| 日常开发 | `speccore execute --task=T-001` | `execute → sync → trace → update` |
| Bug 修复 | `speccore bugfix --name="xxx"` | `bugfix → execute → sync → change` |
| 需求变更 | `speccore impact --req=REQ-001` | `impact → baseline → change → validate` |
| 多人协作 | `speccore progress --detail` | `update --assignee → execute → handover` |
| 长任务链 | `speccore execute --all --batch-size=3` | `execute --resume` |
| CI/CD | `speccore hooks install` | hooks + GitHub Actions 模板 |
| 全量层 | `speccore global-status` | `iteration-from-global → sync-global` |
| 项目迁移 | `speccore migrate --dry-run` | migrate → validate |
| 健康检查 | `speccore health` | `progress → report → dashboard` |
