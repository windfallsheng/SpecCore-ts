## 二、WorkBuddy 集成原理

`speccore init` 在项目初始化时自动创建：

```
my-project/
├── .workbuddy/
│   ├── memory/
│   │   └── MEMORY.md                    # 项目记忆（告知 WorkBuddy Speccore 项目结构）
│   └── skills/
│       └── speccore/
│           └── SKILL.md                 # Speccore Skill（含全部命令和用法）
└── .speccore/                           # Speccore 数据（Spec 文件的唯一真相来源）
```

### MEMORY.md 的作用

告诉 WorkBuddy 这是 Speccore 项目，并标明关键路径：
- `.speccore/` 是 Spec 文件的唯一来源
- `speccore` CLI 是权威工具
- 全局层地址、配置路径等

### SKILL.md（Speccore Skill）的作用

让 WorkBuddy 掌握 Speccore 的全部命令和最佳实践：
- **命令速查表**：按场景分类的所有命令（49 个）
- **多项目全量层**：GLOBAL/ 目录结构说明
- **意图识别**：自然语言匹配规则
- **目录结构**：期次→任务→多端的前端目录规范
- **开发流程**：从导入到执行的完整链路
- **别名速查**：所有命令的快捷别名

## 三、工作流程

```
1. 用户初始化项目
   speccore init
        │
        ├→ 创建 .speccore/ 目录结构
        └→ 创建 .workbuddy/ skill + memory 文件

2. 用户用 WorkBuddy 打开项目
        │
        └→ WorkBuddy 自动加载 .workbuddy/skills/speccore/SKILL.md

3. 用户在 WorkBuddy 中对话
   "帮我创建一个用户登录功能"
        │
        ├→ Speccore Skill 激活
        └→ AI 调用 speccore new-task --name="用户登录"

4. CLI 执行确定性操作
        │
        ├→ 创建 Task 目录结构
        ├→ 生成 Spec 模板文件
        └→ 更新上下文状态
```

## 四、CLI 与 AI 命令的对应

| CLI 命令 | 对应的 AI Slash Command | 说明 |
| :--- | :--- | :--- |
| `speccore init` | `/spec-init` | 项目初始化 |
| `speccore import` | `/spec-import` | 导入项目 |
| `speccore new-task` | `/spec-new-task` | 创建多平台 Task |
| `speccore iteration create` | `/spec-iteration-create` | 创建期次 |
| `speccore execute` | `/spec-execute` | 执行开发 |
| `speccore progress` | `/spec-progress` | 查看进度 |
| `speccore validate` | `/spec-validate` | 合规检查 |
| `speccore impact` | `/spec-impact` | 影响分析 |
| `speccore audit` | `/spec-ai-audit` | AI 审计 |

## 五、更新机制

### 更新 Speccore CLI

```bash
npm update -g speccore
```

### 更新 Skill 文件

```bash
speccore init --force
```

> ⚠️ `--force` 会重新生成所有模板文件。如有定制，先备份。

## 六、安全检查

Speccore CLI 生成的 Skill 文件符合 WorkBuddy 安全规范：
- ✅ 不包含系统命令注入
- ✅ 不访问用户个人文件
- ✅ 仅操作项目目录内的 `.speccore/` 数据
- ✅ 所有文件读写通过 `speccore` CLI 命令完成
