# SpecCore 快速开始

> 📊 [总览](总览.md) | 🚀 最新: v5.18.0 | 💡 直接输入 `speccore` 看自适应引导面板

> 安装 → 初始化 → 导入需求 → 分析 → 拆分 → 开发 → 发布 — 一条龙

## 安装

```bash
npm install -g speccore
```

---

## 核心流程（7 步）

### 1️⃣ 初始化项目

```bash
speccore init
# → .speccore/ + CONSTITUTION + TEAM + 全局计数器
```

### 2️⃣ 导入需求文档

```bash
# 单文件
speccore word2spec -f 后台需求.docx -i Q3 -p 后台管理端

# 批量多端
speccore word2spec --files "admin.docx=后台管理端,h5.docx=后台H5端,web.md=Web" -i Q3
```

### 3️⃣ 需求分析

```bash
speccore analyze --iteration=Q3
# → 完整性扫描 + 源码对标 + 架构影响 → ANALYSIS.md
# → 预览摘要 → [y]保存
```

### 4️⃣ 拆分任务

```bash
speccore iteration split --iteration=Q3
# → 每个 Task 生成 11 份文档 + IMPACT.md + .env.example

# 严格模式（逐节确认）
speccore iteration split --strict
# → [y]确认 [e]改名 [a]分配 [N]跳过 [q]取消
```

### 5️⃣ 执行开发

```bash
# 单任务
speccore execute --task=Task-001 --force --iteration=Q3

# 严格模式（逐项确认）
speccore execute --task=Task-001 --strict --force

# 批量执行
speccore execute --all --force --iteration=Q3

# 跳过指定任务
speccore execute --all --skip=Task-003

# 分批执行（白名单）
speccore execute --all --only=Task-001,Task-002

# 上下文隔离
speccore execute --all --batch-size=3

# 分批 + 跳过
speccore execute --all --batch-size=3 --skip=Task-003 --force

# 分批 + 白名单
speccore execute --all --batch-size=2 --only=Task-001,Task-002,Task-005 --force

# 指定基础分支（依赖任务从上游分叉）
speccore execute --task=Task-002 --base=feature/Task-001-用户管理
```

### 6️⃣ 创建 PR + 合并

```bash
speccore pr --task=Task-001        # 自动创建 PR
speccore merge-check                # 预测合并冲突
speccore lifecycle --task=Task-001 --status=done
```

### 7️⃣ 收尾归档

```bash
speccore done --task=Task-001       # 一键收尾
speccore arch-update --apis=/api/xxx --tables=xxx  # 更新架构
```

## 其他常用操作

### 自适应引导面板

```bash
speccore                   # 根据项目状态自适应显示下一步
# → 💻 开发中 · 3/6 完成 █████░░░░░ 50%
# → 👉 speccore execute --task=Task-004 --force
```

```bash

```

### 宪法自动检测

```bash
speccore constitution       # 扫描项目 → 生成 CONSTITUTION.md 规则
```

### 智能入口（自然语言）



### 智能入口（自然语言）

```bash
speccore spec "新增订单导出功能"    # → 识别为 new_task
speccore spec "修复支付回调超时"    # → 识别为 bugfix
```

### 任务生命周期

```bash
speccore lifecycle                  # 查看状态机
speccore lifecycle --all            # 查看看板
speccore lifecycle --task=Task-001 --status=testing
speccore lifecycle --task=Task-001 --check  # 检查 TEST/REVIEW
```

### 单任务深入分析

```bash
speccore analyze --task=Task-001
# → 预览 TECH/TEST/REVIEW 变更 → [y]确认 → 反复运行幂等
```

### 源码注释标记

```java

// @spec-api POST /api/orders
```

### 回滚

```bash
speccore rollback --task=Task-001 --reason="方案变更"
```

### 查看变更

```bash
speccore tracker     # 全量需求变更追踪
speccore ops         # 命令操作历史
```

## 与 GitHub Spec-Kit 对比

| 维度 | SpecCore | Spec-Kit |
| :--- | :--- | :--- |
| 全链路 | init → rollback（完整闭环） | specify → tasks（到拆分止） |
| 分支管理 | 依赖检测 + 自动分叉 | 无 |
| 质量关卡 | lifecycle 内置强制关卡 | 手动 |
| 自然语言 | 中文意图识别 + AI 降级 | 仅英文 |
| 代码生成 | 模板驱动（可控） | 委派 Copilot/Claude |
| Constitution | CONSTITUTION.md（自动检测+强制） | 强制宪法 gate |

## 改进方向（向 Spec-Kit 学习）

### 1. 代码生成：模板 + 可选外部 Agent

当前 `execute` 用内置模板生成代码。增加 `--agent` 参数：

```bash
# 默认：模板生成（当前行为）
speccore execute --task=Task-001

# 委派外部 AI Agent（已实现）
speccore execute --task=Task-001 --agent=copilot
speccore execute --task=Task-001 --agent=trae
```

优势：保留模板的可控性，同时为高级用户开放外部 Agent。

### 2. Constitution 强制化 ✅ 已实现

analyze 时自动检查 CONSTITUTION.md 规则，不符合时警告提示：
```
📜 宪法规则检查:
  [违反] 宪法要求 RESTful API，但需求中包含 GraphQL
  [提醒] 宪法要求单元测试，请确认 TEST.md 已完善
```

### 3. 命令精简

`speccore`（无参数）→ 显示 19 个核心命令
`speccore --help` → 显示全部 68 个
低频命令折叠到深层 help
