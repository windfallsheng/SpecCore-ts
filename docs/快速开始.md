# SpecCore 快速开始指南

> 从零上手 SpecCore。三种方式效果相同：
> - ⌨️ **终端**: `speccore init`
> - 🤖 **AI Slash**: `/spec-init`（在 WorkBuddy/Cursor 中）
> - 💬 **AI 说话**: "初始化一个项目"

---

## 环境要求

- **Node.js**: >= 18.0.0
- **操作系统**: macOS / Linux / Windows

## 安装

```bash
# 全局安装（推荐）
npm install -g speccore

# 零安装体验（npx）
npx speccore demo     # 直接运行演示项目
npx speccore --help   # 查看帮助

# 验证安装
speccore --version
```

---

## 🎬 零安装体验

不想安装？直接用 `npx` 体验：

```bash
# 运行演示项目（5 分钟）
npx speccore demo

# 在当前目录初始化
npx speccore init
# 💬 "初始化 SpecCore 项目"

# 查看帮助
npx speccore --help
```

## 5 分钟上手

### 第一步：初始化项目

```bash
cd my-project
speccore init
```

初始化自动完成：
- ✅ 创建 `.speccore/` 完整目录结构（含 GLOBAL 全量层）
- ✅ 创建 `.workbuddy/` WorkBuddy 集成文件（skill + memory）
- ✅ 创建 `.speccore/config/platforms.yaml` 多平台配置
- ✅ 更新 `.gitignore`

### 第二步：导入需求文档

```bash
# 批量导入（Word 或 Markdown 格式混着用）
speccore word2spec --files \
  "后台PRD.docx=后台管理端,前端需求.md=Web,小程序需求.docx=H5" \
  -i Q3
# 💬 "把这三个需求文档导入到 Q3"
```

> Word 格式需要 pandoc（命令会提示安装），`.md` 文件直接导入无需 pandoc。

### 第四步：需求分析 + 拆分 + 开发

```bash
# 分析完整性 + 源码对标
speccore analyze

# 拆分（自动端检测 + 风险评分 + 依赖分析）
speccore iteration split

# 开发（严格模式：逐 Task 确认）
speccore execute --task=Task-001 --strict --force

# 查看任务看板
speccore lifecycle --all
```

### 第五步：编辑技术宪法

编辑 `.speccore/CONSTITUTION.md`，填写项目技术栈和命名规范。

```markdown
## 技术栈

### 后端
- 语言：Java / TypeScript / Go
- 框架：Spring Boot / NestJS / Gin
- 数据库：MySQL / PostgreSQL

### 前端
- 框架：Vue / React
- UI 组件：Element Plus / Ant Design
```

### 第三步：导入项目到全量层

```bash
# 导入后端项目
speccore import --project=user-service --path=./backend --type=backend

# 导入前端项目
speccore import --project=frontend-web --path=./web --type=web

# 增量同步模式
speccore import --project=user-service --path=./backend --update
```

### 第四步：创建期次

```bash
speccore iteration create --name 2026-07-用户系统
```

### 第五步：创建多平台任务

```bash
# 多端 Task
speccore new-task --name 用户登录 --platforms=web,h5
# 💬 "创建用户登录任务，Web 和 H5"

# 传统单任务
speccore task new --name 日志审计 --type feature
```

### 第六步：查看进度

```bash
# 整体进度
speccore progress

# 按平台统计
speccore progress --platform=web

# JSON 格式
speccore progress --format json
```

### 第七步：查看任务上下文

```bash
speccore context --task=Task-001
```

## 🧠 智能入口：自然语言

**不想记命令？直接用 `speccore spec` + 自然语言！** 支持 36 种意图类型，200+ 关键词。

```bash
speccore spec "做一个用户登录功能"        # → 自动匹配 goal
speccore spec "开始干活"                   # → 自动匹配 execute
speccore spec "进度怎么样了"               # → 自动匹配 progress
speccore spec "把登录改成验证码登录"        # → 自动匹配 change
speccore spec "全量状态"                   # → 自动匹配 global-status
speccore spec "分析 REQ-001 影响"          # → 自动匹配 impact
```

## 核心场景速查

### 全量层操作

```bash
# 查看全量状态
speccore global-status

# 从全量层生成期次
speccore iteration-from-global --reqs=REQ-001,REQ-002 --name=2026-07-用户系统

# 同步到全量层
speccore sync-global --iteration=2026-07-用户系统

# 重建需求索引
speccore index-update --dry-run        # 预览
speccore index-update                  # 执行
```

### 多端任务

```bash
# 创建多端 Task
speccore new-task --name 用户登录 --platforms=web,h5,miniapp

# 添加新平台
speccore platform-add --name=tablet --description="平板端" --tech="React Native"

# 按平台执行 / 统计
speccore execute --platform=web
# 💬 "开发 Web 端任务"
speccore progress --platform=h5
```

### 开发执行

```bash
# 执行所有待开发任务
speccore execute --all

# 按执行人 / 优先级 / 状态
speccore execute --assignee=张三
speccore execute --priority=high
speccore execute --status=pending

# 预览执行计划
speccore execute --dry-run

# 智能调度方案
speccore plan
```

### 变更与同步

```bash
# 需求变更联动
speccore change --req=REQ-001 --desc="登录增加验证码"

# 代码反向同步
speccore sync --task=Task-001
```

### 分析与审计

```bash
# 变更影响分析
speccore impact --req=REQ-001

# 创建版本基线
speccore baseline create --name=2026-Q3-Release

# 可视化仪表盘
speccore dashboard

# AI 智能审计
speccore audit --strict
```

### 验证与报告

```bash
# 合规性检查
speccore validate --fix

# 生成项目报告
speccore report --format html --output report.html
```

### 维护

```bash
# 重命名
speccore rename --target=旧名 --new-name=新名

# 批量重命名
speccore rename --batch --pattern="Task-" --replacement="Feature-"

# 归档
speccore archive --all

# 生成交接文档
speccore handover --iteration=2026-07-用户系统
```

## 🔧 终端命令速查

> 在终端执行，以 `speccore` 开头。对应在 AI 工具中使用的 Slash Command 以 `/spec-` 开头（如 `/spec-init`、`/spec-new-task`）。

| 命令 | 说明 | AI 命令 |
| :--- | :--- | :--- |
| `speccore init` | 初始化项目（创建目录结构 + WorkBuddy 集成） | `/spec-init` |
| `speccore import` | 导入项目到全量层 | `/spec-import` |
| `speccore new-task` | 创建多平台 Task | `/spec-new-task` |
| `speccore platform-add` | 动态添加平台 | `/spec-platform-add` |
| `speccore iteration create` | 创建期次 | `/spec-iteration-create` |
| `speccore progress` | 进度查看 | `/spec-progress` |
| `speccore execute` | 执行任务 | `/spec-execute` |
| `speccore validate` | 合规性检查 | `/spec-validate` |
| `speccore context` | 查看 Task 上下文 | `/spec-context` |

## 双向追溯：代码 ↔ Spec

在代码中加 `@spec` 注释，SpecCore 自动维护追溯链：

```java
// @spec Task-001-user-login
@RestController
public class AuthController { ... }
```

```bash
# 执行任务（自动创建 Git 分支）
speccore execute --task=Task-001 --force

# 反向同步（扫描 @spec 注释 → 更新 TASK.md）
speccore sync --dry-run     # 预览
speccore sync               # 写入

# 查看追溯链
speccore trace --req=REQ-001
speccore trace --full
speccore current --commit    # 生成 Commit 消息
```
