# ADR-002: Redis 防暴力破解方案

| 属性 | 值 |
| :--- | :--- |
| **状态** | ✅ 已采纳 |
| **决策日期** | 2026-06-29 |
| **决策者** | 架构团队 |
| **影响范围** | 所有涉及认证的 Feature |
| **关联模式** | `anti-brute-force.md` |

## 上下文（Context）

登录接口面临暴力破解风险。需要设计一种机制，在用户体验和安全性之间取得平衡。

## 决策（Decision）

**使用 Redis 实现失败计数 + 自动锁定机制。**

具体设计：
- Key: `login:fail:{phone}`
- Value: 失败次数（整数）
- TTL: 15 分钟（每次失败重置 TTL）
- 阈值: 5 次失败后锁定 15 分钟

## 理由（Rationale）

1. **Redis 高性能**：读写操作在微秒级，不影响登录接口响应时间。
2. **自动过期**：无需额外清理任务，TTL 自动管理。
3. **分布式环境友好**：多实例共享同一 Redis，状态一致。

## 被拒绝的方案（Rejected Alternatives）

| 方案 | 拒绝理由 |
| :--- | :--- |
| **数据库表记录** | 读写频繁，数据库压力大；清理逻辑复杂 |
| **内存缓存（Caffeine）** | 多实例无法共享，无法防止分布式攻击 |
| **无限制策略** | 存在暴力破解风险，不满足安全要求 |

## 影响（Consequences）

- **新增依赖**：必须依赖 Redis 可用性。
- **降级方案**：Redis 不可用时，允许登录但记录 ERROR 日志（风险自担）。

## 降级策略

当 Redis 连接超时或不可用时：
1. 记录 `ERROR` 级别日志。
2. 放行登录请求（不计数、不锁定）。
3. 触发告警通知运维。
