# 项目术语表

> SpecCore 框架专用术语。

## 术语定义

| 术语 | 英文 | 定义 | 使用场景 |
| :--- | :--- | :--- | :--- |
| 期次 | Iteration | 一个开发周期，通常包含多个 Feature | 目录命名、项目管理 |
| Feature | Feature | 一个原子功能模块，可独立开发、测试、上线 | 目录命名、任务分配 |
| 契约 | Contract | 前后端接口的 API 定义，唯一事实源 | API_CONTRACT.yaml |
| 反向同步 | Reverse Sync | 代码与 Spec 冲突时，先更新 Spec 再改代码 | 开发流程 |
| 模式 | Pattern | 可复用的解决方案或踩坑记录 | PATTERNS 目录 |
| ADR | Architecture Decision Record | 架构决策记录 | 记录技术选型原因 |
| AC | Acceptance Criteria | 验收标准 | TASK.md 中定义 |
| 产出物 | Deliverable | 开发完成后必须交付的文件或代码 | TASK.md 清单 |
| 回归 | Regression | 验证已有功能在变更后是否仍正常工作 | 变更管理 |
