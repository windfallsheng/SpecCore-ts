# 项目核心宪法与冲突裁决规则

## 1. 核心原则
- **反向同步铁律**: 代码与 Spec 冲突时，**必须先更新 Spec，再修改代码**
- **Spec 是唯一事实源**: 文档和代码冲突时，错的一定是代码

## 2. 分层总览结构
- 全局层: `.speccore/`
- 期次层: `期次-XXX/00-期次总览/`
- 任务层: `期次-XXX/Task-XXX/`

## 3. 冲突裁决规则
- **L1 - 明确冲突** → 立即修正代码
- **L2 - Spec 缺陷** → 暂停，提交裁决
- **L3 - 环境不可行** → 升级至架构评审

## 4. 代码溯源规范
- 所有生成代码文件必须包含 `@spec` 注释
- `@spec-link` 必须指向对应的 Spec 文件路径

## 5. 规则优先级
1. 本规则文件（最高优先级）
2. `.speccore/CONSTITUTION.md`
3. 口头或对话中的临时指令
