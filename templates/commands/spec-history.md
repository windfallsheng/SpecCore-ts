# /spec-history - 查看需求历史

## 命令说明

输出指定需求（REQ-XXX）的完整变更历史，包括来源、版本演进、关联期次，以及 Git 提交记录。

**可追溯层级**：

| 层级 | 提供什么 | 由谁负责 |
| :--- | :--- | :--- |
| SpecCore 层 | 业务层面的变更原因（为什么要改） | AI 自动生成 Changelog |
| Git 层 | 代码层面的变更记录（谁、何时、改了什么） | 开发者提交 Commit |
| 对话记录 | 决策过程（当时怎么讨论的） | 对话历史存档 |

## 使用方式

### 标准命令格式
```bash
/spec-history --req=REQ-001
```

### 自然语言格式
```bash
查看 REQ-001 历史
REQ-001 变更记录
谁改了用户登录需求
查看用户登录的变更历史
```

## 参数说明

| 参数 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `--req=<ID>` | 是 | 需求 ID | `--req=REQ-001` |

## AI 执行流程

### 第一步：定位需求文件

从 `GLOBAL/INDEX.md` 查找 REQ-001 所在的项目和文件路径。

若未找到：
```markdown
❌ 需求 REQ-XXX 不存在

在 GLOBAL/INDEX.md 中未找到该需求 ID。
请运行 /spec-global-status 查看可用需求列表。
```

### 第二步：提取历史信息

读取 `GLOBAL/PROJECTS/{project}/REQUIREMENT.md`，提取 REQ-001 的：

1. **元数据块**：来源、版本、状态、最后修改、关联信息
2. **变更历史表格**：逐行解析版本演进

### 第三步：补充 Git 日志（可选）

```bash
git log --oneline -10 -- "GLOBAL/PROJECTS/{project}/REQUIREMENT.md"
```
