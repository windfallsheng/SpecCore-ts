# /spec-health - 项目健康度看板

## 命令说明
生成项目健康度报告，包含 Spec 健康度、开发效率、模式复用等维度。

**适用场景**：项目复盘、团队汇报、质量评估

**核心理念**：让数据驱动项目改进。

## 使用方式
```
/spec-health
/spec-health --iteration=2026-07-会议预定
/spec-health --format=json
```

## 参数说明

| 参数 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| 无参数 | - | 查看整体健康度 | `/spec-health` |
| `--iteration=<期次>` | 否 | 查看指定期次 | `/spec-health --iteration=2026-07-会议预定` |
| `--format=json` | 否 | 输出 JSON 格式 | `/spec-health --format=json` |

## AI 执行流程

### 第一步：数据采集

| 数据来源 | 采集内容 |
| :--- | :--- |
| `PROJECT_GRAPH.md` | 任务状态、完成率、依赖关系 |
| 各 `TASK.md` | 产出物状态、变更履历、工时 |
| `.speccore/PATTERNS/README.md` | 模式数量、新增模式 |
| 代码文件 | @spec 注释覆盖率 |
| `ITERATIONS/README.md` | 期次数量、周期 |
| `.speccore/PROJECT/TEAM.md` | 团队成员 |

### 第二步：计算指标

| 维度 | 指标 | 计算方式 |
| :--- | :--- | :--- |
| **Spec 健康度** | Spec 与代码一致性 | 有 @spec 注释的文件 / 总代码文件 |
| | Spec 腐烂率 | 待反向同步 Task / 总 Task |
| | 追溯链完整率 | 有完整追溯链的 Task / 总 Task |
| | 文档完成率 | Spec 完成度 |
| **开发效率** | 平均 Task 完成时间 | 总工时 / 已完成 Task 数 |
| | AI 代码占比 | AI 生成代码行数 / 总代码行数 |
| | 验收通过率 | 一次性通过 AC 的 Task / 总 Task |
| **模式复用** | 模式总数 | PATTERNS 数量 |
| | 模式复用率 | 被引用的模式 / 总模式数 |
| | 最常用模式 | 引用次数最多的模式 |
| **团队健康** | 任务分布均衡度 | 各成员任务数差异 |
| | 成员负载 | 各成员进行中任务数 |
