# /spec-welcome - 首次使用引导

## 命令说明
交互式引导新用户完成第一个完整功能开发，从期次创建到代码交付，全程引导。

**适用场景**：新用户首次使用 SpecCore

**核心理念**：让用户在 5-10 分钟内完成第一个功能，体验完整流程。

## 使用方式
```
/spec-welcome
```

## AI 执行流程

### 第一步：检测首次使用

检测 `.speccore/local/.onboarded` 文件是否存在：
- 若存在：提示「你已完成引导，可直接使用 /spec-goal 开始开发」
- 若不存在：进入引导流程

### 第二步：欢迎与项目检测
```text
👋 欢迎使用 SpecCore！

看起来你是第一次使用这个项目，我来引导你完成第一个功能开发。
整个过程大约需要 5-10 分钟，我会一步步带你完成。

首先检测一下当前项目状态...

{检测结果}
  .speccore/ 目录：✅ 已存在
  技术栈配置：{已配置/未配置}
  团队成员配置：{已配置/未配置}
  现有期次：{N} 个
```

如果 `.speccore/` 不存在，提示用户先运行 `/spec-init`。

### 第三步：期次创建引导
```text
📋 第一步：创建期次

期次是一个开发周期（通常是一个版本或一个迭代）。
请为本次开发输入一个期次名称：

建议格式：{日期}-{项目代号}
示例：2026-07-会议预定

请输入期次名称：
```

用户输入后，AI 执行 `/spec-iteration-create --name=<用户输入>`

### 第四步：需求填写引导
```text
📝 第二步：填写需求

我已经为你创建了期次，并生成了需求文档模板。

请用 1-3 句话描述你要实现的功能：
例如："用户可以通过手机号和密码登录系统，密码错误5次锁定15分钟"

请输入功能描述：
```

用户输入后，AI 将描述填充到 `REQUIREMENT.md` 中。

### 第五步：确认需求
```text
✅ 我已记录你的需求：

【功能名称】：{从描述中提取}
【功能描述】：{用户输入的描述}
【初步识别】：这是一个 {feature/bugfix} 类型的任务

是否按此需求创建第一个任务？
输入 确认 继续，修改 重新描述
```

### 第六步：自动执行（关键决策点）
```text
🚀 第三步：开始开发

我将自动完成以下操作：
  - 拆分任务（/spec-iteration-split）
  - 生成调度方案（/spec-plan）
  - 执行开发（/spec-execute）

执行计划预览：
  - 将创建 1 个 Task
  - 预计生成 3-5 个代码文件
  - 预计耗时 2-5 分钟

是否继续？
输入 确认 开始，查看 查看详细执行计划，取消 终止
```

### 第七步：执行与反馈

执行过程中，AI 实时反馈进度：
```text
⏳ 正在执行...
✅ 任务拆分完成：Task-001-{功能名称}
⏳ 正在生成代码...
✅ 代码生成完成：AuthController.java
✅ 代码生成完成：AuthService.java
⏳ 正在运行测试...
✅ 测试通过：3/3
```

### 第八步：完成与下一步
```text
🎉 恭喜！你的第一个 SpecCore 功能开发已完成！

📊 本次成果：
  - 创建了 1 个 Task：Task-001-{功能名称}
  - 生成了 {N} 个代码文件
  - 通过 {N} 个验收标准
  - 总耗时：{N} 分钟

📋 下一步建议：
  - 运行 /spec-progress 查看整体进度
  - 运行 /spec-goal "下一个功能描述" 继续开发
  - 运行 /spec-demo 体验完整示例项目
  - 运行 /spec-handover 生成交接文档

💡 提示：
  - 所有代码文件都包含 @spec 注释，追溯链已建立
  - 你随时可以运行 /spec-review Task-001 审查产出
  - 后续开发可直接使用 /spec-goal 快速创建功能

{标记 .speccore/local/.onboarded 完成}
```

## 错误处理

| 场景 | 处理方式 |
| :--- | :--- |
| 缺少 `.speccore/` | 提示运行 `/spec-init` |
| 用户中途退出 | 保存进度，下次继续 |
| 执行失败 | 提示错误信息，提供解决方案 |
