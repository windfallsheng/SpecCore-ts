# /spec-ai-audit - AI 智能审计

## 命令说明

AI 自动扫描全量层，发现：
1. **重复需求**：多个项目中是否存在相同或高度相似的需求
2. **歧义描述**：需求描述中模糊的词汇（如"尽量快"、"大量数据"）
3. **潜在冲突**：需求之间存在矛盾
4. **孤立需求**：没有关联任何期次或 Task 的需求

## 使用方式

### 标准命令格式
```bash
/spec-ai-audit
/spec-ai-audit --fix
/spec-ai-audit --detail
```

### 自然语言格式
```bash
智能审计
扫描全量层
检查有没有重复的需求
```

## 参数说明

| 参数 | 必填 | 说明 | 示例 |
| :--- | :--- | :--- | :--- |
| `--fix` | 否 | 自动修复可修复的问题 | `--fix` |
| `--detail` | 否 | 显示详细分析 | `--detail` |

## AI 执行流程

### 第一步：扫描全量层

读取 `GLOBAL/INDEX.md` 和所有 `PROJECTS/*/REQUIREMENT.md`。

### 第二步：智能分析

#### 1. 重复检测

扫描需求名称和描述，计算相似度：
- 相似度 > 80%：🔴 高度重复，建议合并
- 相似度 60-80%：🟡 可能重复，建议审查

#### 2. 歧义检测

扫描需求描述中的模糊词汇：

| 类别 | 模糊词汇 | 建议 |
| :--- | :--- | :--- |
| 时间 | "尽快"、"优先"、"及时" | 指定具体时间或 SLA |
| 数量 | "大量"、"众多"、"若干" | 指定具体数字或范围 |
| 质量 | "良好"、"稳定"、"可靠" | 指定可测量的质量标准 |
| 程度 | "尽量"、"尽可能" | 改为明确的要求 |

#### 3. 冲突检测

扫描需求间的潜在矛盾：
- 技术栈冲突："支持 IE11" vs "仅支持现代浏览器"
- 性能冲突："实时同步" vs "批量同步"
- 权限冲突："所有用户可访问" vs "仅管理员可访问"

#### 4. 孤立检测

找出没有关联任何期次或 Task 的需求。

### 第四步：自动修复（`--fix`）

| 问题类型 | 自动修复方式 |
| :--- | :--- |
| 重复需求 | 在 REQ-021 中添加 `> 已合并至 REQ-001`，更新 INDEX 状态 |
| 歧义描述 | 在需求描述中追加 `⚠️ 待量化：{模糊词汇}` |
| 孤立需求 | 推荐关联期次，等待用户确认 |
