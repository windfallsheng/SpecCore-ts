# 代码与 Spec 冲突裁决规则

## 核心原则
- Spec 是唯一事实源
- 先改 Spec，再改代码

## 三级裁决机制

| 级别 | 场景 | 处理方式 |
| :--- | :--- | :--- |
| **L1 - 明确冲突** | Spec 有定义，代码违反 | AI 立即修正代码 |
| **L2 - Spec 缺陷** | Spec 定义不清或自相矛盾 | AI 暂停并提问 |
| **L3 - 环境不可行** | Spec 方案在当前环境不可行 | AI 升级至架构评审 |

## 裁决升级路径

| 争议级别 | 裁决人 | 时限 |
| :--- | :--- | :--- |
| L1 | AI 自行修正 | 即时 |
| L2 | 产品经理/技术负责人 | 2 小时内 |
| L3 | 架构师/技术委员会 | 4 小时内 |
