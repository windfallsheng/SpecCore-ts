# Task 完成后的维护流程

## 状态定义

| 状态 | 含义 |
| :--- | :--- |
| ✅ 已完成 | Spec 与代码完全对齐 |
| ⚠️ 待反向同步 | 紧急补丁后，代码领先于 Spec |
| 🔄 维护中 | 正在进行回归或修复 |

## 补丁反向同步流程

1. **差异分析**：对比代码与 Spec，生成差异报告
2. **逐项同步**：更新 REQ.md、API_CONTRACT.yaml
3. **更新图谱**：将状态改回 `✅ 已完成`

## Bug 修复流程

1. 标记状态为 `🔄 维护中`
2. 在 TASK.md 中追加问题记录
3. 沉淀到 PATTERNS 踩坑记录
4. 回归验证后恢复状态
