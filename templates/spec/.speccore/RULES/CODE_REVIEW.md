# 代码审查规则

> 本规则定义 `/spec-review` 命令的代码审查检查项。

---

## 一、自动检查项

### 1. 规范遵循

| 检查项 | 严重级别 | 检查方式 |
| :--- | :--- | :--- |
| 代码是否包含 `@spec` 注释 | ❌ 错误 | 文件头部检查 |
| `@spec-link` 是否指向有效的 Spec 文件 | ❌ 错误 | 路径验证 |
| `@spec-type` 是否正确（backend/frontend） | ⚠️ 警告 | 值验证 |
| 是否遵循 `CONSTITUTION.md` 命名规范 | ⚠️ 警告 | 命名模式匹配 |
| 异常处理是否覆盖 Spec 定义的异常场景 | ⚠️ 警告 | 对照 REQ.md |

### 2. 代码质量

| 检查项 | 严重级别 | 检查方式 |
| :--- | :--- | :--- |
| 是否有未捕获的异常 | ❌ 错误 | 异常处理检查 |
| 是否有 SQL 注入风险 | ❌ 错误 | 字符串拼接检查 |
| 是否有硬编码的密码/密钥 | ❌ 错误 | 敏感信息检查 |
| 是否有过长的函数（>100行） | ⚠️ 警告 | 行数检查 |
| 是否有重复代码 | ⚠️ 警告 | 相似度检测 |
| 是否有未使用的导入 | ⚠️ 警告 | 导入检查 |
| 是否有未使用的变量 | ⚠️ 警告 | 变量检查 |

### 3. 测试

| 检查项 | 严重级别 | 检查方式 |
| :--- | :--- | :--- |
| 单元测试覆盖率是否达标 | ⚠️ 警告 | 覆盖率检查 |
| 是否有针对验收标准的测试用例 | ⚠️ 警告 | 对照 AC |
| 是否有边界条件测试 | ⚠️ 警告 | 测试分析 |

### 4. 性能

| 检查项 | 严重级别 | 检查方式 |
| :--- | :--- | :--- |
| 是否有 N+1 查询风险 | ⚠️ 警告 | 数据库调用检查 |
| 是否配置了超时和重试 | ⚠️ 警告 | 配置检查 |
| 是否有内存泄漏风险 | ⚠️ 警告 | 资源管理检查 |

### 5. 安全性

| 检查项 | 严重级别 | 检查方式 |
| :--- | :--- | :--- |
| 是否有 XSS 漏洞风险 | ❌ 错误 | 输入输出检查 |
| 是否有 CSRF 防护 | ⚠️ 警告 | 配置检查 |
| 是否使用安全的加密算法 | ❌ 错误 | 算法检查 |

---

## 二、审查报告格式

```markdown
## 📊 代码审查报告 - Task-001

### 审查信息
- Task：Task-001-用户登录
- 审查时间：2026-07-05 14:30:25
- 审查范围：backend

### ❌ 错误（必须修复）
| 问题 | 位置 | 建议 |
| :--- | :--- | :--- |
| 缺少 @spec 注释 | `AuthController.java:1` | 添加 @spec 注释头部 |
| 硬编码密码 | `AuthService.java:45` | 从环境变量读取 |

### ⚠️ 警告（建议修复）
| 问题 | 位置 | 建议 |
| :--- | :--- | :--- |
| 函数过长 | `AuthService.java:80-150` | 拆分为多个小函数 |
| 覆盖率 72% | `AuthControllerTest.java` | 补充边界条件测试 |

### ✅ 通过项
- 命名规范：通过
- 异常处理：通过
- @spec-link：通过
- SQL 注入检查：通过

### 📋 评分
| 维度 | 得分 | 说明 |
| :--- | :--- | :--- |
| 规范遵循 | 3/5 | 缺少 @spec 注释 |
| 代码质量 | 4/5 | 有 1 个长函数 |
| 测试 | 2/5 | 覆盖率不足 |
| 安全性 | 5/5 | 无安全问题 |
| 性能 | 4/5 | 无性能问题 |
| **综合评分** | **3.6/5** | |

### 💡 改进建议
1. 🔴 优先修复硬编码密码问题
2. 🔴 补充 @spec 注释
3. 🟡 提升测试覆盖率至 90%
4. 🟡 拆分 AuthService 中的长函数
```
